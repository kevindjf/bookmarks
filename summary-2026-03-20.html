Paperclip + Claude : une entreprise zero humain sur votre machine

@RoundtableSpace
Voir le tweet original
Resume

Paperclip est un outil qui, couple a Claude, permet d'installer et faire tourner une "entreprise automatisee" entierement geree par IA sur votre ordinateur local, sans intervention humaine.

Utilite

Montre jusqu'ou l'automatisation par agents IA peut aller : gestion de taches business completes (marketing, support, operations) sans humain dans la boucle. Utile pour prototyper des workflows autonomes ou tester des idees business rapidement.

Comment utiliser

Installer Paperclip, le connecter a Claude, definir les agents et leurs roles (vente, support, contenu...). L'outil orchestre les agents pour executer les taches de maniere autonome.

Mise en place

Telecharger Paperclip depuis le lien dans la video, configurer l'acces a Claude (cle API ou Claude Desktop), definir les workflows souhaites. Commencer par un cas simple (ex: generation de contenu automatique) avant de scaler.

Regles de workflow orchestration pour Claude Code

@roeymiterany (en reponse a @melvynx)
Voir le tweet original
Resume

Partage de regles extraites pour orchestrer Claude Code efficacement : activer le mode plan pour toute tache non triviale (3+ etapes), re-planifier immediatement si quelque chose deraille, structurer le travail en phases.

Utilite

Fournit un framework concret pour eviter que Claude Code parte dans la mauvaise direction sur des taches complexes. Le reflexe "stop and re-plan" est particulierement precieux.

Comment utiliser

Integrer ces regles dans votre fichier `CLAUDE.md` ou vos skills personnalises. Forcer le mode plan (`/plan`) des qu'une tache depasse 3 etapes.

Mise en place

Ajouter dans votre `CLAUDE.md` les regles de workflow : plan mode par defaut, re-planification si blocage, validation avant execution. Tester sur un projet existant pour calibrer.

Mode OpusPlan : Opus pour planifier, Sonnet pour executer

@CamilleRoux
Voir le tweet original
Resume

En utilisant `/model opusplan` dans Claude Code, on peut configurer Opus (plus puissant) pour la phase de planification et Sonnet (plus rapide/economique) pour l'execution du code.

Utilite

Optimise le rapport qualite/cout : la reflexion architecturale beneficie de la puissance d'Opus, tandis que l'execution repetitive de code est geree par Sonnet, plus rapide et moins cher.

Comment utiliser

Taper `/model opusplan` dans Claude Code. La bascule est automatique entre les deux modeles selon la phase.

Mise en place

Immediat. Lancer Claude Code, taper `/model opusplan`, et commencer a travailler. Comparer les resultats avec votre mode habituel pour evaluer le gain.

Pre-commit hook avec Claude Code : simplify, security, tests et doc

@YannDecoopman
Voir le tweet original
Resume

Configuration d'un hook pre-commit dans Claude Code qui execute automatiquement : `/simplify` et `/security-review` avant chaque commit, lance les tests (unitaires + navigateur) avec rollback si echec, et met a jour la documentation/Obsidian si necessaire. Le tout en agents paralleles.

Utilite

Automatise le controle qualite a chaque commit. Empeche d'envoyer du code non revise, non teste ou non documente. Le pattern agents en parallele accelere le processus.

Comment utiliser

Configurer des hooks dans les settings Claude Code qui declenchent les skills `/simplify` et `/security-review`, puis les tests, avant chaque commit.

Mise en place

1) Creer les skills `/simplify` et `/security-review` si pas deja presents. 2) Configurer un hook pre-commit dans `settings.json` de Claude Code. 3) Ajouter les commandes de test dans le hook. 4) Configurer la mise a jour doc/Obsidian en post-action.
